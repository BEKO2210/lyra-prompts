WAS IST BELKIS ONE?
═══════════════════════════════════════════════════════════════

Belkis One ist ein globales Echtzeit-Analyse-Dashboard das den Zustand der Welt in einem einzigen, atemberaubenden Scroll-Erlebnis visualisiert. Keine KI-Analyse — reine Daten, realer Code, echte Quellen. Die Website scrapet öffentliche Datenquellen, verarbeitet sie durch Algorithmen und zeigt ein Gesamtbild: Stehen wir am Abgrund — oder entwickeln wir uns weiter?

Das Herzstück: Ein monumentaler WELT-INDIKATOR — ein Schieberegler der sich aus hunderten von Datenpunkten berechnet und auf einer Skala von "Kollaps" bis "Goldenes Zeitalter" anzeigt wo die Menschheit gerade steht.

Jeder Scroll ist ein Erlebnis. Jede Animation hat Bedeutung. Wenn der User unten ankommt, will er sofort wieder nach oben scrollen — weil es so gut ist.

SCROLL-ENGINE — Das Herzstück:

Die Scroll-Engine basiert auf diesen modernen APIs:
1. ScrollTimeline API (CSS Scroll-Driven Animations) wo verfügbar
2. IntersectionObserver als Fallback für Section-Triggers
3. requestAnimationFrame für smooth interpolierte Werte
4. CSS scroll-snap-type für sektionsweises Snapping (optional togglebar)

Implementierung:
- Jede Section hat data-attributes: data-section="akt-1" data-progress="0"
- Die Engine berechnet pro Frame die Scroll-Position relativ zur Section
- Dieser Wert (0.0 - 1.0) wird an alle Visualisierungen der Section gepusht
- Visualisierungen nutzen diesen progress-Wert für ihre Animationen
- GPU-beschleunigte Properties: transform, opacity, clip-path, filter
- KEINE Layout-Properties animieren (kein width, height, top, left)
- will-change sparsam einsetzen, nur auf aktive Sections

PARTIKEL-SYSTEM:

Canvas-basiert mit diesen Features:
- Pool von 2000 Partikeln (recycled, nie neue erstellt)
- Jeder Partikel hat: x, y, vx, vy, life, color, size, opacity
- Physik-Simulation: Leichte Gravitation + Turbulenz (Perlin Noise)
- Farbwechsel basierend auf aktuellem Akt (passend zum Farbschema)
- Mouse-Interaktion: Partikel weichen dem Cursor aus (Repulsion-Force)
- Performance: OffscreenCanvas + Web Worker wenn verfügbar
- Fallback: Reduzierte Partikelanzahl auf schwächeren Geräten
- requestAnimationFrame mit delta-time für gleichmäßige Animation
